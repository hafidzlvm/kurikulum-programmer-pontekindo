OOP, Object Oriented Programming adalah sudut pandang bahasa pemrogramman yang berkonsep "Object".

    1. Object adalah data yang berisi field / properties / attributes dan method / function / behavior
    Properties dan Method dua hal ini adalah bahasa yang sering di gunakan di dalam OOP, adapun kita lebih mengenal nya dengan variable dan function.

    2. Class adalah blueprint, prototype atau (bahasa awamnya adalah) cetakan untuk membuat Object. Class itu deklarasi nya berisikan semua properties dan functions yang dimiliki oleh Object.
    Setiap object selalu dibuat dari class
    Class ini nanti nya akan di pakai pada sebuah object
    Sebuah class bisa membuat object tanpa batas

    Class itu yang umum nya, suatu benda. Dan object adalah varian nya, contoh benda laptop lalu memiliki object msi, asus, thinkpad, dan lain-lain 

Jenis-Jenis Penulisan
    1. Snake_Case // Disambung dengan garis underline
    Ini_Adalah_Contoh_Dari_Snake_Case();
    2. CamelCase // CamelCase semua suku kata menyatu dan terdapat huruf kapital sebagai pemisahnya
    IniAdalahContohDariCamelCase();
    3. KebabCase // Pemisahnya adalah tanda strip
    Ini-Adalah-Contoh-Dari-Kebab-Case();
    4. ALLcaps // Semua Capital
    IniAdalahContohDariAllCaps();

Encapsulation bisa di gambarkan seperti smartphone dari beberapa data tersebut ada yang bisa di akses ada juga yang tidak bisa, Encapsulation menyebabkan data tersebut tersembunyi dari pihak luar, bukan menghilangkan data tersebut.

Abstractoin adalah mekanisme saat proses dalam sebuah objek disembunyikan, jenis ini mirip dengan jenis yang Encapsulation. Contoh abstraction di dunia nyata bisa kita lihat ketika ingin mengendarai mobil. Kita hanya perlu menekan pedal gas jika ingin mobil berjalan tanpa harus tau apa yang telah terjadi ketika pedal gas tersebut ditekan. Ini berlaku juga ketika kita ingin melakukan pengereman.

Polymorphism merupakan kemampuan objek, variabel, atau fungsi yang dapat memiliki berbagai bentuk. Secara umum polymorphism dalam OOP terjadi ketika suatu SuperClass direferensikan ke dalam SubClass. Alhasil kita dapat mengembangkan sebuah program secara umum, bukan spesifik.
